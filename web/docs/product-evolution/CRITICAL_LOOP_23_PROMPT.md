# Critical Loop 23 — 从静态助手到可继续的任务运行时

## 评委批评

当前助手会展示任务结构，但缺少 Codex 式的持续执行体验：

- 缺少信息被错误显示为“未完成”或“失败”；
- 用户只能重新发起任务，不能在原任务中补充并继续；
- 工作台预览只是文字清单，看不到页面实际会怎样变化；
- 配置型任务显示“0 项工具”，容易让用户误以为什么都没做；
- 工作台模块可以移动，但显示方式仍被固定，首页也尚未由同一模块系统渲染；
- 深色主题只替换局部颜色，浅色 surface 混入后导致大面积灰白和低对比。

## 实现提示词

请把 Agent 改造成可持续的任务运行时，而不是一次性问答：

1. 增加 `awaiting_input`，严格区分等待输入、等待确认、执行失败、完成和取消。
2. 将缺失信息转换为最多三项结构化输入；能用选项时不要求用户写长文。
3. 增加 continue API，在原 task id 内保留已完成步骤并重新规划后续步骤。
4. 生成真实的工作台 before/after 预览：模块位置、宽度、新增/隐藏和工作流均需可视化。
5. 只有真正执行异常才显示“失败”；配置任务无需金融工具时明确说明原因。
6. Dashboard Module Registry 增加兼容可视化类型；允许在仪表、表格、折线、K 线、矩阵、列表和数字卡中选择。
7. 不兼容的模块与图表组合必须被 schema 拒绝。
8. 重做 dark theme token 和所有受影响 surface，不得通过混入白色生成深色页面。
9. 最终将首页改为由同一 Module Registry 和持久化布局渲染，使编辑结果真正改变主工作台。

## 验收

- ETF 重合任务缺少代码时显示“等待补充”，不是失败；
- 用户逐项回答后同一任务继续，历史中不产生重复死任务；
- 修改前后预览能在不读文字说明的情况下识别新增、隐藏与布局变化；
- 确认后首页实际变化，刷新后保持；
- 用户可把风险模块切换为仪表、暴露模块切换为矩阵；
- 深色主题下正文、次要文字、控件、图表和禁用态均清晰可读；
- 任一失败都停在具体步骤，并提供针对性恢复动作。
